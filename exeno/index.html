exeno

anonymous connection app that allows users to privately select contacts from their address book, discovering mutual connections between two people without premature disclosure of identities.

core concept

the premise of exeno is to facilitate connections between individuals who have a pre-existing, real-world relationship. a user can privately select a contact from their own address book. identities are only revealed to both parties if a mutual selection is made, resulting in a "match."

key features

  • anonymous selections: make a selection from your contact list without revealing your identity
  • contact list integration: the app securely reads the device's contact list to enable users to make their selections
  • mutual match reveals: identities are only revealed when two users have mutually selected each other
  • secure backend: built on supabase for robust user authentication, data storage, and serverless functions
  • user-initiated sms invitations: to notify a contact that they have been selected, the app facilitates sending a one-time sms invitation

sms flow

to ensure compliance and a positive user experience, the sms notification system follows a strict, user-initiated process:

  1. user selects a contact: the user navigates their contact list within the app and chooses a person to select
  2. explicit user consent: before any message is sent, the app presents a confirmation dialog
  3. user confirms action: the user must affirmatively tap "confirm" to authorize the sending of the sms
  4. one-time invitation sent: a single, anonymous sms is sent to the recipient
  5. opt-out included: every invitation sms includes mandatory language to allow the recipient to opt out

tech stack

  • frontend: react native with expo
  • state management: zustand
  • backend-as-a-service: supabase (authentication, postgresql database, storage, edge functions)
  • language: typescript

status

exeno is currently under active development. the core features have been implemented, and the project is in the process of being configured for a production environment.
